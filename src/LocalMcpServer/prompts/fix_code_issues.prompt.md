## 검사 항목

1. **버그**: null 참조, 범위 초과, 리소스 누수, 경쟁 조건, 무한 루프 등
2. **안티패턴**: 빈 catch 블록, 매직 넘버, God 클래스, 깊은 중첩 등
3. **보안**: SQL 인젝션, XSS, 하드코딩된 비밀번호, 안전하지 않은 역직렬화 등
4. **성능**: 불필요한 할당, O(n²) 루프, 비효율적 문자열 연결 등
5. **모범 사례**: using/dispose 누락, async void, ConfigureAwait 등
