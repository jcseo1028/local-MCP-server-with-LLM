## 분석 지시사항

위 데이터를 기반으로 아래 항목을 작성하세요. 추측이 아닌 데이터에 근거한 분석만 작성하세요.

### 1. 프로젝트 개요
- 프로젝트명, 주 언어, 파일 수 등 기본 정보
- 디렉터리 이름과 클래스명으로 추론한 프로젝트의 목적과 도메인

### 2. 아키텍처 레이어
- 디렉터리를 기준으로 레이어/모듈을 구분
- 각 레이어의 책임을 타입 선언 기반으로 설명
- 레이어 간 의존 방향을 상속/구현 관계에서 추론

### 3. 핵심 타입 분석
- 가장 중요한 클래스/인터페이스 5~10개를 선정
- 각 타입의 역할, 상속 관계, 다른 타입과의 연결 설명
- interface가 있다면 어떤 구현체가 있는지 연결

### 4. 데이터/제어 흐름
- 진입점(Main, Program, Startup 등)에서 시작하여 데이터가 흐르는 경로 추적
- 주요 유스케이스 1~2개의 흐름을 단계별로 설명

### 5. 설계 패턴 및 특징
- 사용된 설계 패턴(DI, Factory, Observer 등)이 있으면 식별
- 프로젝트의 강점과 개선 가능한 영역

파일 경로와 클래스명을 반드시 포함하여 구체적으로 작성하세요.
