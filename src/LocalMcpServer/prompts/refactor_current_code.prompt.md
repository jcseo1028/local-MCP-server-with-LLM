## 리팩터링 규칙

1. **가독성 향상**: 변수명·메서드명을 의미 있게 개선한다.
2. **중복 제거**: 반복되는 코드를 메서드/함수로 추출한다.
3. **구조 개선**: 단일 책임 원칙(SRP)을 적용한다. 너무 긴 메서드는 분리한다.
4. **현대적 문법**: 해당 언어의 최신 관용 표현(idiom)을 사용한다.
   - C#: pattern matching, collection expression, target-typed new 등
   - Python: f-string, walrus operator, dataclass 등
   - JavaScript/TypeScript: optional chaining, destructuring, nullish coalescing 등
5. **동작 보존**: 기존 기능과 동작을 변경하지 않는다 (리팩터링만).
6. **과도한 변경 금지**: 핵심 구조만 개선하고, 불필요한 리팩터링은 하지 않는다.
7. **들여쓰기를 원본과 동일하게 유지한다** — 원본이 탭이면 탭, 스페이스면 스페이스를 사용한다. 들여쓰기 크기도 원본과 동일하게 한다.
