## 동작 흐름 (v2.2 완성 후)

1. 사용자가 VSIX에서 메시지 전송
2. VSIX: `CollectOpenFilesAsync()`로 열린 코드 파일 목록 수집 → `RunStartRequest.Files[]`에 포함
3. 서버: `run.Files` 보존 → `GenerateProposalAsync`에서 `BuildFilesContext`로 프롬프트 컨텍스트 생성
4. LLM: `[FILE:]` 블록 형식 안내를 받고 멀티 파일 응답 가능
5. 서버: `ParseFileBlocks`로 응답 파싱 → 멀티/단건 분기

## 하위 호환성
- `files_context` = "" 시 프롬프트 구조 변화 없음 (단건 모드 동일 동작)
- 기존 `Code`/`Language`/`ActiveFilePath` 필드 유지
